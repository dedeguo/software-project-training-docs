    🚀 第一篇：校园图书管理系统带练¶

    用一个完整项目，走完 AI 协同开发的第一程¶

    这一篇学什么

    在真正开始自己的项目之前，先完成一次“小而完整”的软件开发演练。

    你将以校园图书管理系统为载体，体验一个项目从认识业务、阅读文档、使用 AI 协作开发，到测试、部署和提交成果的全过程。重点不是一次写完所有代码，而是建立一套以后可以复用的工作方法。

    完成第一篇后，你将拥有

    一个能运行的图书管理系统实践项目、一份可复查的测试记录，以及一套“先读文档、再拆任务、边做边验证”的 AI 协作习惯。

    从系统体验开始 查看项目模板仓库

    🎯 这一篇要帮你做到¶

    看得懂 会协作 做得出 能交付
    理解用户、管理员、图书和借阅之间的业务关系 用 Trae、Superpowers-zh 与 AGENTS.md 约束 AI 行为 在模板基础上完成一个可验证的功能模块 完成测试、部署、Git 提交并能说明自己的成果

    本篇遵循下面的开发路径：

    先体验成品 → 再理解工具和规则 → 阅读需求与设计 → 分步开发 → 真实测试 → 部署验证 → 留下提交记录

    🗺️ 你将完成的 5 个小节¶

    小节 主要任务 本节成果
    1.1 系统体验 体验用户端与管理端，理解项目目标、角色和业务规则 知道系统要解决什么问题、各角色能做什么
    1.2 AI 工具 认识 Trae 与 Superpowers-zh，建立工程化 AI 协作意识 能在项目中安装并使用 AI 工程方法框架
    1.3 AI 规则 理解 Rules、Skills、MCP 与 AGENTS.md 的分工 一份约束 AI 行为的项目规则
    1.4 分步开发 基于图书管理系统模板，阅读文档、拆分任务、实现功能 一个经过验证的图书管理模块
    1.5 测试与部署 导入 Postman 集合、测试界面、填写报告、部署 Tomcat 测试证据、测试报告与可访问系统

    先完成闭环，再追求完整

    这一篇不要求你一下做完所有模块。先把“登录 → 查询图书 → 借阅或归还 → 查看结果”中的一条业务流程做通、测通、说清，再逐步补充其他功能。

    📦 开始前需要准备什么¶

    类别 建议准备
    开发工具 Trae、Git、JDK 17、Maven、MySQL、Tomcat、Node.js
    AI 协作 Superpowers-zh、项目级 AGENTS.md
    测试工具 Postman、Chrome 或 Edge
    项目材料 图书管理系统模板、需求分析说明书、系统设计说明书、测试报告模板

    课程提供的图书管理系统模板包含项目骨架、文档、原型、Postman 测试集合和测试报告模板。请在自己的工作目录中克隆或下载后再开始：

    打开模板仓库 查看需求与设计文档

    不要跳过文档直接让 AI 写代码

    需求分析说明书决定系统“做什么、怎样验收”；系统设计说明书说明“如何分层、接口怎样设计、数据如何保持一致”。先读文档，才能判断 AI 的方案是否符合项目事实。

    🤝 本篇统一使用的 AI 协作流程¶

    每完成一个小任务，都按同一套流程推进：

    1. 提供上下文：让 AI 先阅读 AGENTS.md、需求文档、设计文档和相关已有代码；
    2. 确认方案：要求 AI 说明当前现状、实现思路、影响范围和准备修改的文件；
    3. 分步实现：一次只完成一个可以验证的小任务；
    4. 运行验证：检查代码、启动项目、操作接口和页面；
    5. 记录问题：出现错误时提供复现步骤、预期结果、实际结果和日志；
    6. 审查与提交：确认改动符合规则，完成测试后创建 Git 提交。

    明确任务 → 提供上下文 → AI 提出方案 → 人工审核 → 分步实现 → 运行测试 → 修正问题 → Git 提交

    AI 不是项目代写者

    AI 可以帮助你读代码、提出方案、生成初稿和分析错误，但不能替你确认需求、运行系统或对结果负责。任何你不能运行、不能解释、没有验证过的内容，都不算有效成果。

    ✅ 完成本篇的最低验收标准¶

    完成第一篇时，请对照下面的清单自查：

    • 能说明普通用户和管理员各自能做什么；
    • 已在项目中使用 Trae，并配置或阅读 AGENTS.md；
    • 已阅读需求分析说明书和系统设计说明书；
    • 已完成至少一个可运行、可验证的功能模块或业务闭环；
    • 已导入 Postman 集合并执行关键接口测试；
    • 已在浏览器中完成用户端和管理端的主要操作测试；
    • 已填写测试结果或缺陷记录；
    • 已打包部署，或至少完成项目启动和关键流程验证；
    • 已创建能说明成果的 Git 提交；
    • 能解释一个核心业务规则，例如借阅上限、库存变化或权限校验。

    🧗 可以挑战到什么程度¶

    学习层次 建议目标
    保底目标 模板跑起来，完成登录、图书查询和一个核心操作的测试
    标准目标 完成借阅与归还闭环，处理库存、权限和异常场景，并部署到 Tomcat
    进阶目标 补充自动化测试、优化交互体验、完善测试报告，或用 MCP 辅助调试与验收

    基础薄弱也不必急着追求复杂架构。先完成一个小而完整、可以展示的闭环，比堆出一堆还不能运行的功能更有价值。

    🚦 从这里开始¶

    1. 体验校园图书管理系统，先认识最终要完成的项目；
    2. 配置 Trae 与 Superpowers-zh，准备 AI 协作工具；
    3. 建立项目 AI 规则，明确 AI 开发红线；
    4. 基于模板分步开发，完成一个可验证模块；
    5. 测试并部署系统，把开发成果变成可交付作品。

    先看见成果，再理解过程；先跑通一个闭环，再扩展完整系统；先用证据证明完成，再进行下一步。
