## 三、教学理念与能力培养

随着 AI 编程工具的发展，软件开发方式正在发生变化。本课程不再单纯训练学生编写代码，而更加关注：

- 软件工程过程能力；
- AI 协作开发能力；
- 问题分析与需求建模能力；
- 系统设计与技术决策能力；
- 软件质量保障能力。

> 相关文件见 [teaching-plan/AI时代《软件开发综合项目实训》课程目标与能力培养.md](teaching-plan/AI时代《软件开发综合项目实训》课程目标与能力培养.md)。

### 1. 软件工程实践能力

学生能够：

- 分析真实问题；
- 完成项目选题；
- 编写需求文档；
- 进行系统设计；
- 完成开发、测试和部署。

### 2. AI 辅助开发能力

学生能够：

- 使用 AI 编程工具辅助开发；
- 编写有效 Prompt；
- 使用 Rules、Skills、MCP 等机制约束 AI；
- 理解 AI 生成内容并进行验证。

### 3. 工程思维能力

学生能够：

- 根据需求进行技术选型；
- 权衡开发成本和系统复杂度；
- 建立文档、代码、测试之间的对应关系。

## 四、教学项目案例体系（project-cases）

该目录用于存放课程教学项目案例。

> 当前文件见 [project-cases/《软件开发综合项目实训》教学项目案例体系与使用方案.md](project-cases/《软件开发综合项目实训》教学项目案例体系与使用方案.md)。

项目案例不是简单展示代码，而是用于支撑完整的软件工程教学流程。每个案例建议包含：

```text
项目案例
│
├── 项目背景
├── 用户与业务问题
├── 项目目标
├── 需求分析文档
├── 系统设计文档
├── 数据库设计
├── 核心代码
├── 测试报告
├── 部署说明
└── AI辅助开发过程
```

### 1. 教师课堂演示

教师通过完整项目展示软件项目全过程：

```text
发现问题 → 项目选题 → 需求分析 → 系统设计 → AI辅助开发 → 测试部署 → 成果展示
```

### 2. 学生项目参考

学生可以参考项目规模、文档结构、功能设计、技术方案和开发流程，但不直接复制项目代码。

### 3. 项目评价参考

教师可以依据案例建立评价标准：

- 需求是否清晰；
- 设计是否合理；
- 功能是否完整；
- 测试是否充分；
- AI 使用是否规范；
- 项目成果是否可展示。

## 八、维护说明

本目录主要由课程负责人维护。新增课程资料时，请按照以下原则：

1. 教学文件（大纲、实施方案）放入 `teaching-plan/`；
2. 项目案例资料放入 `project-cases/`；
3. 教学研究与改革探索放入 `teaching-research/`；
4. 教学参考笔记放入 `notes/`；
5. 学生使用资源放入 `docs/`（电子教程）或 `docs/doc-templates/`（文档模板）；
6. AI 工具和技能放入 `ai_skills/`。

保持教师资源、学生资源、AI 工具三类内容相互独立，便于长期维护和扩展。
