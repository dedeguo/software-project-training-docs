# 16周项目开发路线

| 周次 | 主题 | 主要成果 |
|---|---|---|
| 第1周 | 课程导入与AI编程体验 | 环境检查、示例项目修改 |
| 第2周 | 项目选题 | 选题与可行性说明 |
| 第3周 | 需求分析 | 用户角色、功能清单、业务流程 |
| 第4周 | 原型与总体设计 | 页面原型、系统架构 |
| 第5周 | 数据库设计 | ER图、数据字典、建表SQL |
| 第6周 | 接口设计与项目骨架 | 接口清单、可运行骨架 |
| 第7周 | 登录与权限 | 用户认证和角色控制 |
| 第8周 | 核心业务模块一 | 第一个完整业务闭环 |
| 第9周 | 核心业务模块二 | 主要业务规则 |
| 第10周 | 前后端联调 | 页面、接口和数据联通 |
| 第11周 | AI或特色功能 | AI功能或项目特色功能 |
| 第12周 | 测试与缺陷修复 | 测试用例、缺陷清单 |
| 第13周 | 工程质量与安全 | 校验、异常、日志、安全检查 |
| 第14周 | 部署上线 | 可访问的部署地址 |
| 第15周 | 项目包装 | README、报告、视频、PPT |
| 第16周 | 项目答辩 | 演示、代码抽查、现场修改 |

## 每周课堂节奏

1. 教师讲解与示范；
2. 本周任务拆解；
3. 小组项目开发；
4. 阶段展示与提交。

!!! warning "不要拖到最后几周才开始"
    项目成绩不仅取决于最终效果，还要检查阶段成果、Git提交记录、AI协作记录和成员贡献。
