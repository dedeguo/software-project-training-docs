## 一、 获取带练项目资源

当你对系统的最终形态有了清晰的概念后，请在本地终端执行以下命令，将项目的初始骨架克隆到你的工作目录：

```bash
# 克隆校园图书管理系统带练仓库
git clone https://github.com/dedeguo/book-manage.git

# 进入项目目录
cd book-manage

```

## 二、 认识标准工程目录

克隆完成后，**请使用 Trae 打开该项目目录**。一个规范的、对 AI 友好的工程结构如下所示，请务必先熟悉它们的位置：

```text
book-manage/
├── docs/                  # 核心！项目文档库（Trae 生成代码的唯一准则）
│   ├── requirement.md     # 需求说明：规定系统“要做什么”
│   ├── design.md          # 详细设计：规定系统“怎么做”
│   └── database.sql       # 数据库初始化脚本
├── frontend/              # Vue 前端工程目录
├── backend/               # Spring Boot 后端工程目录
├── AGENTS.md              # AI 行为约束规则（我们稍后将配置它）
└── README.md              # 项目快速启动说明

```

## 三、 体验的第一步：把本地系统跑起来

现在你已经见识过线上的完整版了，接下来你要让本地的“半成品”也跑起来。请查阅项目根目录的 `README.md`，完成本地 MySQL 的数据库初始化（执行 `docs/database.sql`），并分别启动前后端服务。

当你在本地 `localhost` 也能看到与线上体验地址一样的初始登录界面时，意味着你已经准备好进入真正的 AI 辅助开发实战了。
