# 认识并运行示例项目

本节以校园图书管理系统为例，练习阅读项目资料、运行工程和理解基本结构。此时不要求掌握所有代码，重点是建立对完整软件项目的直观认识。

## 学习目标

- 能根据 README 准备运行环境；
- 能说明系统面向谁、解决什么问题；
- 能找到前端、后端、数据库和配置文件；
- 能运行系统并完成一次基本业务操作；
- 能记录运行问题及解决过程。

## 先了解项目

课程示例：[校园图书管理系统](https://github.com/dedeguo/book-manage)

运行代码前，先阅读项目介绍、需求和设计资料，并回答：

1. 系统有哪些用户角色？
2. 系统管理哪些核心数据？
3. 借书、还书等核心业务如何流转？
4. 前端、后端和数据库分别承担什么职责？
5. 项目需要哪些软件和运行参数？

!!! tip "先形成整体认识"
    不要一开始逐行阅读代码。先看项目目标、页面、业务流程和目录结构，再带着问题定位代码。

## 运行项目

按照项目 README 完成以下操作：

1. 克隆或下载代码；
2. 创建数据库并执行初始化脚本；
3. 检查数据库连接、端口等本地配置；
4. 安装项目依赖；
5. 启动后端和前端；
6. 登录系统并完成一次查询或借阅操作；
7. 保存运行截图和关键日志。

## 理解工程结构

尝试在项目中找到：

- 项目启动入口；
- 页面与路由；
- Controller、Service 和数据访问代码；
- 数据库脚本；
- 环境配置文件；
- 构建与依赖配置；
- README 和其他项目文档。

选择一个页面，从页面操作出发，追踪它调用的接口、业务逻辑和数据表。不要求记住所有代码，但要能够描述一次请求的大致路径。

## 使用 AI 辅助阅读

```text
请先阅读当前项目的README和目录结构，不要修改代码。

请说明：
1. 项目的主要用户和核心业务；
2. 前端、后端、数据库分别位于哪里；
3. 项目的启动步骤和必要配置；
4. 【指定页面或功能】对应的主要代码文件；
5. 仍需要我人工确认的信息。

引用具体文件作为依据，不要根据文件名猜测不存在的功能。
```

AI 的说明只能作为阅读线索，必须通过代码、配置和实际运行结果进行核对。

## 提交成果

- 示例项目运行截图；
- 项目结构说明；
- 一条核心业务的代码定位记录；
- 运行问题及解决记录。

## 验收检查

- [ ] 项目能够正常启动；
- [ ] 能演示至少一个真实业务操作；
- [ ] 能说明主要目录的职责；
- [ ] 能找到一个页面对应的接口和数据表；
- [ ] 配置文件中没有提交真实密码或密钥。
